## Part 1：Agent 的定义

`Agent` 不是一次性的问答工具，而是一个**持续运转的闭环系统**。给定一个任务，它自己决定如何搜索信息、调用工具、生成输出、读写记忆、进行反思——每一步都不是预设脚本，而是模型根据当前状态**当场判断**的。

### 闭环框架

Agent 的核心循环可以概括为：

**感知 → 规划 → 行动 → 观察 → 记忆 →（回到感知）**

```
        ┌──────────┐
        │   感知    │
        └────┬─────┘
             ↓
        ┌──────────┐
        │   规划    │←──────────────┐
        └────┬─────┘               │
             ↓                     │
        ┌──────────┐          ┌────┴─────┐
        │   行动    │          │   记忆    │
        └────┬─────┘          └────▲─────┘
             ↓                     │
        ┌──────────┐               │
        │   观察    │───────────────┘
        └──────────┘
```

### 五个环节

**感知（Perception）**——把环境状态读入上下文。

在 LLM Agent 中，这通常意味着：用户输入、上一步工具的返回值、代码执行结果等，被**拼接进当前 context**，成为模型下一步决策的依据。

**规划（Planning）**——Agent 的「灵魂」。

基于当前状态，决定「接下来做什么」。典型流程是 `提出候选 → 评估 → 选择`，从多个可能的动作中挑出一个执行。

**行动（Action）**——执行被选中的动作，分两类：

- **外部行动**：调用工具 / API、操作网页、控制物理设备等
- **内部行动**：推理、读写记忆——注意，**推理本身也是一种行动**

**观察（Observation）**——行动改变了环境，Agent 收到新的反馈（ground truth）。

据此判断任务进展，决定是否需要纠错、重试或调整计划。

**记忆（Memory）**——跨步骤、跨会话保留信息：

- **工作记忆（Working Memory）**：当前上下文
- **长期记忆（Long-term Memory）**：经验（episodic）、知识（semantic）、技能（procedural）

### Agent 与 Pipeline 的本质区别

<aside class="callout">
<p><strong>关键点</strong>：在这个循环里，「选择下一步动作」是<strong>模型当场做出的判断</strong>，而不是写死的 <code>if/else</code> 分支。正是这种动态决策，构成了 Agent 与传统 Pipeline 的根本差异。</p>
</aside>

Pipeline 的路径在部署时就固定了：先检索、再生成、再格式化——无论输入是什么，走同一条路。Agent 则面对不同任务会走出**完全不同的执行路径**：有的任务需要先搜资料，有的直接写代码，有的要反复试错。同一个 Agent，循环可以跑很多轮，直到任务完成或达到终止条件。
